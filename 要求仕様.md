# 大学受験向け学習Webアプリ 仕様書

## 1. 文書の目的

本書は、大学受験向けの英単語・地理などの問題を、スマートフォンのWebブラウザで学習できる「mikan風」学習アプリの実装仕様を定義するものである。

本仕様書は、Claude Code でそのまま実装を進めやすいように、以下を明確にする。

* システム構成
* 画面仕様
* データ仕様
* 学習ロジック
* 保存方式
* 実装優先順位
* 受け入れ条件

---

## 2. システム概要

### 2.1 アプリの目的

大学受験対策として、英単語・地理などの問題を、スマホでテンポよく学習できるWebアプリを提供する。

### 2.2 想定利用者

* 3人程度
* 各自が自分のスマートフォンで使用する
* ログイン機能は不要
* セキュリティは重視しない

### 2.3 基本方針

* スマホ向けのシングルページWebアプリとする
* 無料で運用可能な構成とする
* 問題データは Google スプレッドシートで管理する
* 学習履歴は各端末のブラウザ内に保存する
* 間違えた問題を繰り返し学習できることを重視する

---

## 3. 推奨システム構成

### 3.1 構成

* フロントエンド: React
* 配信環境: GitHub Pages または Cloudflare Pages
* 問題データ管理: Google スプレッドシート
* 問題データ取得: 公開JSONまたはGoogle Apps Script経由のJSON API
* 学習履歴保存: localStorage

### 3.2 推奨理由

* サーバー不要で無料運用しやすい
* 問題編集をスプレッドシートで簡単に実施できる
* 3人程度の利用であれば性能的にも十分
* 学習履歴は個人ごとに端末へ保持できる

### 3.3 将来拡張余地

将来的に以下へ拡張可能な構成とする。

* IndexedDB への切替
* Supabase 等への進捗保存
* PWA対応
* オフライン対応

## 7. 問題データ仕様

### 7.1 問題データ管理方針

問題データは Google スプレッドシートで管理する。
1行を1問として管理する。

### 7.2 スプレッドシート列定義

以下の列を基本とする。

| 列名          | 必須 | 内容            |
| ----------- | -- | ------------- |
| id          | 必須 | 問題ID。一意であること  |
| subject     | 必須 | 教科名。例: 英単語、地理 |
| unit        | 必須 | 単元名           |
| question    | 必須 | 問題文           |
| choice1     | 必須 | 選択肢1          |
| choice2     | 必須 | 選択肢2          |
| choice3     | 必須 | 選択肢3          |
| choice4     | 必須 | 選択肢4          |
| answer      | 必須 | 正解の選択肢番号。1〜4  |
| explanation | 任意 | 解説            |
| enabled     | 任意 | true のものだけ出題  |
| order       | 任意 | 並び順           |

### 7.3 JSONデータ形式

アプリ内では次の形式で扱う。

```ts
export type Question = {
  id: string;
  subject: string;
  unit: string;
  question: string;
  choices: string[];
  answerIndex: number; // 0-3
  explanation?: string;
  enabled?: boolean;
  order?: number;
};
```

### 7.4 サンプル

```json
[
  {
    "id": "eng_0001",
    "subject": "英単語",
    "unit": "基本単語",
    "question": "apple の意味は？",
    "choices": ["りんご", "机", "学校", "車"],
    "answerIndex": 0,
    "explanation": "apple は りんご を意味する。",
    "enabled": true,
    "order": 1
  }
]
```

## 8. 学習履歴データ仕様

### 8.1 保存先

* localStorage
* キー名は `studyHistory` とする

### 8.2 保存目的

* 間違えた問題の再出題
* 連続正解数の管理
* 学習状況の可視化

### 8.3 データ形式

```ts
export type StudyHistoryItem = {
  correct: number;
  wrong: number;
  lastResult: "correct" | "wrong" | null;
  streak: number;
  lastAnsweredAt: string | null;
};

export type StudyHistory = Record<string, StudyHistoryItem>;
```

### 8.4 保存例

```json
{
  "eng_0001": {
    "correct": 2,
    "wrong": 5,
    "lastResult": "wrong",
    "streak": 0,
    "lastAnsweredAt": "2026-04-15T16:20:00"
  },
  "geo_0010": {
    "correct": 4,
    "wrong": 1,
    "lastResult": "correct",
    "streak": 3,
    "lastAnsweredAt": "2026-04-15T16:30:00"
  }
}
```

## 9. 学習ロジック仕様

### 9.1 正誤更新ルール

問題回答時に以下を更新する。

#### 正解時

* `correct += 1`
* `lastResult = "correct"`
* `streak += 1`
* `lastAnsweredAt = 現在日時`

#### 不正解時

* `wrong += 1`
* `lastResult = "wrong"`
* `streak = 0`
* `lastAnsweredAt = 現在日時`

---

### 9.2 出題モード

#### 通常学習

* 単元内の全問題を対象とする
* 未学習問題も出題する
* ただし、苦手問題を優先して出題する

#### 苦手問題のみ

以下のいずれかを満たす問題を対象とする。

* `lastResult === "wrong"`
* `wrong > 0 && streak < 3`
* 未学習ではない

#### 未学習のみ

* 履歴が存在しない問題のみを出題する

---

### 9.3 苦手問題優先ロジック

通常学習時は次の優先順位で並び替えて出題する。

1. 直近で間違えた問題
2. 不正解回数が多い問題
3. 連続正解数が少ない問題
4. 未学習問題
5. それ以外

具体的には、各問題に対してスコアを計算し、降順で並び替えてもよい。

例:

* `lastResult === "wrong"` なら +100
* `wrong * 10`
* `streak * -5`
* 未学習なら +20

実装は簡易でよいが、「間違えた問題が繰り返し出やすい」ことを満たすこと。

---

### 9.4 習得済み判定

以下を満たす問題は一旦「習得済み」とみなして出題頻度を下げる。

* `streak >= 3`

ただし、完全に除外する必要はない。
通常学習時には低頻度で出てもよい。

## 19. mikan 参照仕様まとめ

本アプリは mikan そのものを再現することを目的とせず、学習体験の参考として公開情報から読み取れる要素を整理し、大学受験向け4択学習アプリへ適用する。

### 19.1 公開情報から読み取れる mikan の特徴

公開されている公式情報・ストア情報では、mikan は以下の特徴を持つ。

* 英単語・英熟語・リスニング・文法・長文など幅広い英語学習に対応している
* 大学受験、英検、TOEIC、TOEFL、IELTS、GRE など複数の試験用途に対応している
* 発音付きで単語学習ができる
* 1語3秒、10語30秒のような高速学習を訴求している
* 多肢選択学習やカードをめくる形式の学習がある
* 習熟度に応じて適した単語を出し分けるアルゴリズムを持つとしている
* 覚えていない単語を繰り返し復習できる
* 学習記録の蓄積・可視化機能がある
* 継続学習を促す演出や応援要素がある

### 19.2 今回のアプリで参考にする点

本仕様では、mikan のうち以下を参考要素として採用する。

1. テンポのよい1問1画面学習
2. 短時間で連続回答できる操作性
3. 間違えた問題の繰り返し学習
4. 学習履歴の保存と可視化
5. スマホで片手操作しやすいUI
6. 学習対象を単元ごとに選べる構成

### 19.3 今回のアプリで採用しない点

初期実装では以下は対象外とする。

* 有料プラン
* 出版社教材連携
* 英語以外の総合機能拡張
* リスニング専用教材群
* 長文読解や模試形式
* 辞書連携
* キャラクター演出
* 通知、連続学習日数、ランキング

### 19.4 mikan 参照を踏まえたUX要件

本アプリは mikan 風の軽快さを重視し、以下を満たすこと。

* 回答操作は1タップで完了できる
* 回答後すぐに正誤が分かる
* 次の問題に素早く進める
* 苦手問題は再出題されやすい
* 1セッションを短く回せる
* 学習完了時に達成感がある

### 19.5 mikan 参照を踏まえた追加推奨機能

必須ではないが、余力があれば追加を検討する。

* 正解時・不正解時の簡易アニメーション
* 連続正解表示
* 学習継続回数の表示
* 単語カードモード
* 音声再生ボタン

### 19.6 実装判断の方針

Claude Code は、mikan 風という曖昧な表現をそのまま実装せず、以下のように具体化して判断すること。

* mikan 風 = スマホでテンポよく進む軽量な学習体験
* mikan 風 = 苦手問題の反復に強い
* mikan 風 = 難しい操作説明が不要
* mikan 風 = 1回の学習開始までの操作数が少ない

したがって、UI は過度に複雑にせず、学習開始までの階層を浅く保つこと。
