# 問題出力規則 仕様書

## 目的

本仕様書は、学習アプリで扱う問題データのCSV形式、生成規則、画面表示規則、出題順、履歴反映規則をまとめたものです。

対象は主に英単語問題です。現在の実装では、Googleスプレッドシートの `questions` シート、または生成済みCSVを読み込み、アプリ内で出題用データへ変換します。

## 問題データの基本形式

問題データはCSVの1行を1問として扱います。

必須または主要な列は以下です。

| 列名 | 内容 |
| --- | --- |
| `id` | 問題ID。例: `leap_ja_0001`, `leap_en_0001` |
| `subject` | 教科。例: `英語` |
| `unit` | Unit名。例: `英単語 LEAP 和訳` |
| `largeCategory` / `term` | 大分類。どちらかが使われる |
| `middleCategory` / `range` | 中分類。どちらかが使われる |
| `question` | 問題文 |
| `choice1` ～ `choice8` | 選択肢 |
| `answer` | 正解選択肢番号。1始まり |
| `explanation` | 解答後に表示する解説 |
| `tags` | タグ。複数可 |
| `difficulty` | 難易度。数値 |
| `enabled` | `false` の場合は出題しない |

読み込み時の補完規則:

- `largeCategory` がなければ `category1`, `majorCategory`, `term`, `semester` を順に参照します。
- `middleCategory` がなければ `category2`, `minorCategory`, `range`, `scope` を順に参照します。
- `answer` はCSV上では1始まり、アプリ内部では0始まりに変換します。
- `enabled` が `false` の行は除外します。
- 選択肢は空欄を除外し、正解選択肢が存在しない問題は除外します。

## ID規則

IDは以下の形式を推奨します。

```text
<教材>_<方向>_<4桁番号>
```

例:

```text
leap_ja_0001
leap_en_0001
target1900_ja_0001
```

`_0001` のように末尾がアンダースコア + 数値の場合、リスト表示や問題画面でその番号を表示できます。

## 問題形式

### 和訳問題

英単語を表示し、意味を選択します。

例:

```text
question: 次の英単語の意味として最も適切なものを選びなさい: consider
choice1: ～をよく考える
choice2: ...
answer: 1
```

IDまたはUnitに `_ja_` または `和訳` が含まれる問題は、問題表示時に英単語音声の自動読み上げ対象になります。

### 英訳問題

意味を表示し、英単語を選択します。

例:

```text
question: 次の意味に最も近い英単語を選びなさい: ～をよく考える
choice1: consider
choice2: ...
answer: 1
```

### 直接入力問題

対象問題の正解が英語テキストであり、英単語問題として判定できる場合、出題条件画面で「直接入力」を選べます。

直接入力の判定条件:

- Unit、ID、大分類などに英単語系の識別子が含まれる
- 正解選択肢に英字が含まれる
- 正解選択肢が日本語ではない

直接入力時の判定は以下を正規化して比較します。

- 前後空白を除去
- 小文字化
- アポストロフィ表記を統一
- ハイフン表記を統一
- 連続空白を1つに統一

## 選択肢規則

CSVには最大8個の選択肢を持たせます。

出題時には以下の処理を行います。

1. CSV上の正解選択肢を保持します。
2. 不正解選択肢から最大3個をランダムに選びます。
3. 正解1個 + 不正解最大3個をシャッフルします。
4. 表示時の `answerIndex` を再計算します。

つまり、CSV上では正解を常に `choice1` にしても問題ありません。実際の出題では選択肢順が毎回シャッフルされます。

## 解説・語源メモ規則

解説は `explanation` に記録します。

語源情報がある場合は、解説末尾に語源メモとして追記します。

語源情報の優先順位:

1. `語源.csv`
2. gogengoなど外部語源データ
3. 語源情報なし

`語源.csv` に該当語がある場合は、その情報を優先します。

## 分類規則

### 大分類

大分類は主に出題範囲の大きなまとまりとして使います。

例:

- 学期
- 範囲A～E
- 教材内のセクション

問題選択画面では、大分類ごとに以下を表示します。

- 正答済み数 / 問題数
- 進捗ゲージ

大分類の前回選択値は端末に保存します。

### 中分類

中分類は大分類内のさらに細かい範囲として使います。

大分類が選択されている場合は、その大分類内に存在する中分類だけを表示します。

### TAGと難易度

TAGと難易度は現在画面上では非表示です。

ただし、実装上の状態・絞り込み処理は残しています。将来復活させる場合は、表示フラグを戻すことで再表示できます。

## 出題条件画面の対象問題

問題選択画面では、以下の条件に一致する問題だけを対象にします。

- 教科が一致する
- Unitが一致する
- 選択中のTAGに一致する
- 選択中の難易度に一致する
- 選択中の大分類に一致する
- 選択中の中分類に一致する

TAG・難易度は現在非表示ですが、内部条件としては残っています。

## 出題数

通常の学習セッションでは最大25問を出題します。

対象問題が25問未満の場合は、対象問題数だけ出題します。

## 出題モード

### おすすめ

対象問題をスコア順に並べ、上位から出題します。

スコアが高くなる要素:

- 復習期限が来ている
- 復習期限を過ぎている
- 苦手度が高い
- 間違い回数が多い
- ためらい回数が多い
- 遅答回数が多い
- 直近が遅答
- 平均回答時間が遅い
- streakが低い

スコアが下がる要素:

- 復習期限がまだ来ていない
- mastered扱いになっている

同点時は、基本的に元データの順番が維持されます。

### ランダム

対象問題を重み付きランダムで出題します。

出やすくなる要素:

- 未実施
- 苦手
- 遅答
- 間違いが多い
- streakが低い

出にくくなる要素:

- 正答率が高い
- mastered扱い
- 直近も正解している

重み付き抽選は重複なしで行います。

### 復習

復習期限 `nextReviewAt` が今日以前の問題だけを対象にします。

並び順はおすすめスコア順です。

### 苦手

苦手判定された問題だけを対象にします。

苦手判定条件:

- `manualWeak` が true
- 直近結果が wrong
- `weakWeight` が一定以上
- ためらい回数が多い
- 間違いがあり、streakが一定未満

並び順はおすすめスコア順です。

### 遅答

遅答判定された問題だけを対象にします。

遅答判定条件:

- 直近が遅答
- 遅答回数が一定以上
- 平均回答時間がしきい値以上

並び順はおすすめスコア順です。

### 未学習

履歴がない問題だけを対象にします。

## リスト表示規則

出題条件画面では、対象問題をリスト表示できます。

リスト表示の規則:

- ID順で表示します。
- ID末尾が `_0001` 形式の場合、左側にその数値を表示します。
- 問題をクリックすると、同じ行の右側に正解を表示します。
- 苦手問題、遅答問題は色分けします。
- 苦手設定は横方向のスライドトグルで変更できます。

## 問題画面表示規則

問題画面では以下を表示します。

- 現在のセッション内番号
- セッション総数
- 選択中の大分類内での番号
- ID末尾番号がある場合は `No. 0001` の形式で表示
- 問題文
- 選択肢または直接入力欄

問題文の長さによって文字サイズを調整します。

| 文字数 | 表示クラス |
| --- | --- |
| 45文字以下 | short |
| 46～90文字 | medium |
| 91～150文字 | large |
| 151文字以上 | xl |

## 音声読み上げ規則

和訳問題では、問題表示時に英単語を読み上げます。

音声規則:

- 英語音声を優先します。
- `en-US` の音声を最優先します。
- なければ `en-*` の音声を使います。
- 消音ボタンで自動読み上げを止められます。
- 再生ボタンで手動再生できます。

## 回答後表示規則

回答後には以下を表示します。

- 正解 / 不正解
- マスコット画像
- 解説
- 回答時間
- 正誤結果
- 次へボタン
- 終了ボタン

正解時で回答時間が遅い場合は、「迷った」扱いで進めるボタンを表示します。

## 履歴更新規則

回答ごとに問題履歴を更新します。

更新される主な項目:

- `correct`
- `wrong`
- `lastResult`
- `streak`
- `bestStreak`
- `mastered`
- `reviewLevel`
- `intervalDays`
- `ease`
- `weakWeight`
- `nextReviewAt`
- `firstAnsweredAt`
- `lastAnsweredAt`
- `answeredCount`
- `quickCount`
- `slowCount`
- `lastSlow`
- `hesitatedCount`
- `responseTimeTotalMs`
- `weakReducedAt`

端末内の `localStorage` には回答ごとに即時保存します。

Googleスプレッドシートへは、差分またはsnapshotとしてバックアップします。

## 復習間隔規則

正解時は復習レベルを上げ、次回復習日を先に延ばします。

不正解時は以下を行います。

- reviewLevelを0に戻す
- intervalDaysを0にする
- weakWeightを増やす
- nextReviewAtを今日にする
- masteredをfalseにする

苦手度 `weakWeight` の減少は、短時間で下がりすぎないように制限します。

現在の規則:

- 前回の苦手度減少から10時間以上経過している場合のみ、正解時に `weakWeight` を下げる
- 減少時刻を `weakReducedAt` に保存する

## CSV生成規則

### LEAP

`public/generate_leap_vocab_csv.py` で生成します。

出力ファイル:

- `questions_leap_english_all.csv`
- `questions_leap_japanese_all.csv`
- `questions_leap_all.csv`

生成規則:

- 和訳問題ID: `leap_ja_0001`
- 英訳問題ID: `leap_en_0001`
- 正解はCSV上では `choice1`
- 不正解選択肢は同一リスト内の別項目から生成
- 出題時に選択肢をシャッフルする
- 語源メモは `語源.csv` を優先する

### Target1900 / 共通重要語

`scripts/generate_full_vocab_csv.py` で生成します。

出力ファイル例:

- `questions_target1900_generic_all.csv`
- `questions_target1900_translation_generic_all.csv`
- `questions_common_important_generic_all.csv`
- `questions_vocab_generic_all_sources_with_translation.csv`

生成規則:

- 和訳問題ID: `<prefix>_ja_0001`
- 英訳問題ID: `<prefix>_0001`
- 正解はCSV上では `choice1`
- 不正解選択肢は同一教材内の別項目から生成
- 出題時に選択肢をシャッフルする

## 変更時の注意

問題出力規則を変更する場合は、以下を確認します。

1. CSV列名がアプリの読み込み処理と一致しているか
2. `answer` が1始まりで正しく設定されているか
3. 正解選択肢が空欄でないか
4. `enabled=false` の問題が意図せず含まれていないか
5. ID末尾番号が重複していないか
6. 和訳問題のIDまたはUnitに `_ja_` / `和訳` が含まれているか
7. 出題時シャッフル後も正解が正しく再計算されるか
8. 履歴の `questionId` とCSVの `id` が一致しているか
